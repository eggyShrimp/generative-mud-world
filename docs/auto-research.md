# 04 — Auto Research 与组织形式演化

## 设计原则

不预设社会形态（王国/共和国/部落/教团），**从 Entity 的属性和历史中涌现出组织形式**。

LLM 不预设技术树节点，**基于世界状态即时生成候选方案**。

---

## 组织形式演化

### 核心思路

不是设计固定形态树（部落 → 城邦 → 王国...），而是**每个 Entity 面临自身问题，LLM 生成针对性方案，组织内部权力结构投票采纳**。

### 演化流程

```
问题检测(规则) → LLM 生成候选方案 → 权力结构投票 → 采纳/否决 → 更新 Entity 能力
```

- **规则层**检测治理问题（规模超出治理能力、外部威胁、内部不满、经济衰退等）
- **LLM** 生成 2-4 个候选方案（每种方案有名称、叙事、优劣势、解锁行为）
- **规则层**通过权力中心投票决定采纳（LLM 只管"菜单"，NPC 负责"点菜"）
- 采纳中的内部斗争本身就是叙事内容

### 关键

Entity 的 `governanceForm`、`identityLabel` 等字段是自然语言字符串，由 LLM 在形态演化时生成。`deriveAvailableActions` 用关键词匹配 + 阈值组合判断可用性，而非枚举判断。

---

## Auto Research：全局知识演化

不只是组织形态，整个世界的知识体系也自生长。研究节点动态插入研究树，后续研究可以以此为前置，形成真正的技术演进路径而非预设的技能树。

### 研究触发机制

当某个领域的活动积累超过阈值时，触发 LLM 生成该领域的下一个合理研究。新研究节点被插入研究树，解锁新的行为或能力。

---

## 语言演化

语言和文化是同一枚硬币的两面——当文化分化时，语言也跟着分化。不是预设"通用语/精灵语/矮人语"，而是**每种方言从 simulation 的历史中自然生长出来**。

### 方言自生长路径

| 路径 | 触发条件 | 产出 |
|------|----------|------|
| 地理隔离 | 两区域交流稀疏 > N 月 | 方言分化（词汇替代 + 惯用语） |
| 行会/阶层 | 某职业 NPC 密度超阈值 + 持续交流 | 行话（特定群体内部用语） |
| 重大事件 | 世界级事件 scope 覆盖 + 持续超阈值 | 事件流行语（沉淀进方言） |
| 派系形成 | 派系建立 | 派系专用词汇 |

### 传播与代际

- 俚语通过社交图传播，与谣言系统复用同一套传播网络
- 年轻 NPC 更倾向用新词，年老 NPC 保留旧词——语言本身承载代际差异
- 方言作为对话生成的横切面：注入 LLM prompt 的 style modifier，自动应用词汇替换和风格修饰

## 统一演化框架

语言、宗教、法律、军事、艺术——所有社会模块遵循同一套演化模式。不是为每个模块设计独立的演化系统，而是抽取它们的**共同骨架**。

### 骨架：五阶段流水线

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  1. 检测      │ → │  2. 生成      │ → │  3. 裁决      │ → │  4. 物化      │ → │  5. 传播      │
│  Detector     │    │  Generator    │    │  Arbiter      │    │  Materializer │    │  Propagator   │
│              │    │              │    │              │    │              │    │              │
│ 规则检测条件  │    │ LLM 生成候选  │    │ 权力/随机/    │    │ 把采纳的候选  │    │ 通过社交图/   │
│              │    │              │    │ 共识决定采纳  │    │ 写入世界状态  │    │ 区域扩散      │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
                                                                    │
                                                                    ▼
                                                          ┌─────────────────┐
                                                          │  成为新一轮输入   │
                                                          │  (影响后续生成)  │
                                                          └─────────────────┘
```

LLM 只在阶段 2（生成）介入。其余四个阶段都是规则层，不需要 LLM。

### 各模块裁决策略

不同模块用不同裁决逻辑：

- **权力投票** — 组织、法律（权力中心投票决定）
- **阈值扩散** — 流行语、俚语（扩散超阈值自动采纳）
- **事件驱动** — 军事（战争胜败证明战术有效性）
- **随机起点** — 艺术风格（随机事件触发）

### 各模块演化一览

| 模块 | 现有池（规则维护） | 演化产出（LLM） | 裁决方式 |
|------|-------------------|----------------|----------|
| 语言 | 方言 + 词汇 | 新方言/行话/俚语 | 阈值扩散 |
| 宗教 | 教义 + 仪式 | 教义分裂/新崇拜 | 信众人数决定 |
| 法律 | 法条 + 判例 | 新法律/改革法案 | 权力投票 + 反抗风险 |
| 军事 | 战术 + 兵制 | 新战术/军事传统 | 战争胜败 |
| 艺术 | 流派 + 作品 | 新风格/杰作 | 传播广度 |
| 经济 | 商品 + 合约类型 | 新货币/新交易方式 | 采用者数量 |
| 建筑 | 风格 + 结构 | 新风格 | 造价 + 材料 |
| 教育 | 知识树 | 学派/理论/发现 | 知识扩散率 |
| 阶层 | 阶级定义 | 新阶层/身份标签 | 经济 + 权力结构 |
| 社交 | 关系标签 | 新习俗/礼仪/禁忌 | 传播 + 共识 |

### LLM 的角色限定

在整条流水线中，LLM 只做一件事：基于模块现有池和触发原因，生成 2-4 个候选的"下一个新东西"。规则层接管裁决、物化、传播。

**LLM 永远不决定"是否发生演化"，只负责"如果发生演化，可以演化成什么"。** 演化本身由规则驱动，LLM 只是让演化结果的**叙事多样性**远超人手工创建的内容池。
