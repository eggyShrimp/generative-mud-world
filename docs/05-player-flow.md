# 05 — 玩家接入与反馈

> 本文档补充 `00-architecture.md` 中未展开的玩家接入流程细节。

## 玩家接入流程

```
1. 连接 → 世界简介 + 可选 Entity 列表

   可选 Entity 来源:
     - 初始化时预设的"玩家可选角色"
     - 世界中自然生成的"空位" (NPC 死亡 → 继承人无 AI)
     - 创建新角色: 在某城镇创建普通 NPC，从此起点开始

2. 选择 → 中途接入时，LLM 生成 onboarding 摘要:
   "你接管的这个角色是西境铁匠老铁。最近他的学徒跑了，
    新开的武器铺抢了生意，但他帮隔壁寡妇修农具在镇上口碑不错..."

3. 随时可切换实体 (有限制):
     - 个体 → 只能切到有关系或继承权的实体
     - GM/observer 模式 → 可观察任何实体但不能操作
```

### 死亡/退出

```
个体角色死亡 → 有继承人 → 自动切换
             → 无继承人 → 选择: 新建角色 / 接管现有 / observer

组织被消灭 → 同样回到选择界面
```

## 反馈闭环

```
玩家行动 → SimulationDelta + 涟漪评估
         → 世界状态改变
         → 受影响 NPC 行为改变
         → 新事件产生
         → AOI 过滤 → 推送到玩家终端
         → 玩家看到结果 → 下一个行动
```

**任务日志已实现**（`app.tsx:1495` QuestPanel — 列表/详情/追踪/放弃），但**无主线剧情**。反馈不来自预写剧本，而来自 simulation 中其他 NPC 的规则响应 + LLM 叙事化。NPC 通过对话 `offer_quest` 工具提供任务，任务进度由 `engine/quest-tracker.ts` 实时追踪。

## 信息流

### AOI 过滤

玩家看到的信息 = 当前控制实体的"视野"半径内的世界事件。

### 日报

每回合结算后，为每个玩家生成个性化日报：
- 叙事摘要（LLM 生成）
- 数值变化
- 世界新闻（AOI 范围内的事件）
