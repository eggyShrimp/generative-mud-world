# 02 — 模拟层详细设计

> 本文档补充 `00-architecture.md` 中未展开的模拟层细节。

## NPC 分层激活

大规模世界中，不可能给每个 NPC 调 LLM。按激活范围分层：

```
图层        数量     激活方式                   LLM 消耗
──────────────────────────────────────────────────
核心 NPC    ~100    全量 LLM: 感知→记忆→决策    高
区域 NPC    ~2000   仅在玩家/核心 NPC 附近才激活  中
背景 NPC    ~8000   schedule 模板驱动，不需要 LLM  零
```

**核心公式**：`LLM 调用量 ≈ 玩家交互次数 + 重大世界事件 × 受影响核心 NPC 数`

### 变化注入：LLM 不推 NPC，LLM 推世界

```
确定性规则 + 规则输入持续变化 = 不可预测的涌现行为
```

**Simulation 是物理引擎，LLM 是 Dungeon Master。** DM 不需要指挥每个 NPC，只需要偶尔注入世界事件，让涟漪在规则系统中扩散。

## 多人接入：Entity 控制模型

### 设计原则

不给玩家预分"角色类型"。玩家控制任意 Entity，该 Entity 的可用能力由其当前属性动态决定。

### 可用命令由属性推导

Entity 的可用命令由其当前属性动态推导：人口 ≥ 10 可组建团体，认可度 ≥ 50 可提议结盟，武力 + 凝聚力达标可宣战，财富 + 人口达标可征税，影响力达标可颁布法令。推导逻辑是关键词匹配 + 阈值组合，不依赖枚举类型。

`governanceForm`、`identityLabel` 等是自然语言字符串，由 LLM 在形态演化时生成。

### 组织形式涌现

```
属性积累 (成员/财富/武力/影响力/认同) →
  规模跃迁 + 权威跃迁 + 治理跃迁 →
    不预设路径，LLM 在阈值触发时动态生成
```

详见 `04-auto-research.md`。

### 多人并发模型

```
                    World State (单例)
                         │
         ┌───────────────┼───────────────┐
         │               │               │
    ┌────▼────┐    ┌─────▼──────┐   ┌────▼────┐
    │ Player A │    │ Player B   │   │ Player C│
    │ (控制城市)│    │ (控制商人) │   │ (控制国家)│
    └────┬────┘    └─────┬──────┘   └────┬────┘
         │               │               │
    ┌────▼────────────────▼───────────────▼────┐
    │           Command Queue                   │
    │  所有玩家命令排队，按回合统一处理           │
    │  每回合:                                  │
    │    1. 处理玩家命令                         │

    └───────────────────────────────────────────┘
```

## AOI (Area of Interest) 事件过滤

AOI 过滤规则：每个玩家只能看到其控制实体所在区域的事件、全局事件，以及符合特定条件的跨区域事件（如高影响力实体可见外交事件）。不同控制对象看到不同范围：

| 控制类型 | 可见 | 不可见 |
|----------|------|--------|
| 冒险者 | 房间事件 → 城镇事件 → 世界公告 | 邻国密谋、远方战争 |
| 王国 | 本国外交/军事/内政 → 邻国公开行动 → 世界公告 | 邻国密谋细节 |
| 商人行会 | 贸易路线 → 商品价格 → 竞争行会公开行动 | 非贸易政治细节 |

## NPC 记忆系统

### 记忆压缩的激活条件

```
触发记忆压缩 = 该 NPC 满足以下任一条件:
  1. 与玩家/核心 NPC 有过直接交互（近期活跃）
  2. 经历了重大世界事件（事件影响范围内）
  3. 积累的新观察超过阈值 N 条 (从 llmTriggerConfig.minMemoriesToTrigger 读取)

不满足 → 不调 LLM，记忆按时间线性追加
```

### 规模估计

```
10000 NPC 世界中:
  ├── 与玩家同区域活跃的:    ~50 NPC   → 可能触发压缩
  ├── 受世界事件波及的:      ~200 NPC  → 可能触发压缩
  └── 纯背景 NPC:          ~9750 NPC → 不压缩，记忆自然丢弃
```

### 背景 NPC 记忆策略

```
保留最近 7 天的观察，超时自动丢弃
不压缩，不反思，不产生长期人格变化
→ 玩家走近时，基于 schedule 和当前状态做出反应
→ 如果玩家深度互动，才"升级"为核心跟踪 NPC
```

**只有被玩家"看见"或"互动过"的 NPC，才值得消耗 LLM 去做人格演化。**
