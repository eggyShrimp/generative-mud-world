# TUI 交互规范

本文定义终端客户端的玩家交互规则。修改 TUI 前先检查本规范，再检查 `tui-style.md`、`tui-typography.md` 和 `tui-conventions.md`。

## 目标

TUI 是文字冒险 / MUD 游戏界面，不是管理后台。玩家每一屏都应该清楚知道：

- 我在哪里
- 这里有哪些出口
- 眼前有哪些目标
- 当前能做什么
- 刚才的操作产生了什么结果

## 完成标准

一次 TUI 修改只有同时满足以下条件，才算完成：

- 当前地点是主视觉，房间名和描述优先显示。
- 出口始终可见，不被目标菜单、背包、对话、等待提示遮挡。
- 目标交互遵循“先选目标，再选动作”。
- 通用操作位置固定，不和目标动作混在一起。
- 选中目标不会改变主布局高度。
- 每次操作后都有事件日志反馈。
- `Esc` 在所有临时状态中都表示关闭当前状态或返回上一层。
- 宽屏和窄屏都能看清地点、出口、目标、日志和通用操作。

## 信息层级

### 第一层：常驻信息

常驻信息包括：

- 当前地点
- 出口
- 眼前目标
- 事件日志
- 角色状态
- 通用操作

这些信息不能被普通目标菜单遮挡。背包和对话可以出现临时面板，但也不能遮住出口和事件日志。

### 第二层：上下文操作

选中 NPC、物品或其他可交互对象后，显示该目标的可用动作。动作只来自当前目标，不展示无关操作。

### 第三层：独立流程

只有背包、对话、等待中这类流程可以进入独立面板。它们仍然必须保持返回规则清楚。

## 布局规则

### 宽屏

宽屏指终端宽度至少 100 列。宽屏使用左主右辅、底部日志布局：

```text
┌────────────────────────────────────────────┬────────────────────┐
│ 当前地点                                    │ 角色状态             │
│ 房间名                                      │ 需求条               │
│ 房间描述                                    │                    │
│                                            │ 行动                 │
│ 出口                                        │ [R] 休息             │
│ [N] 北  [E] 东                              │ [Q] 状态             │
│                                            │ [I] 背包             │
│ 眼前                                        │ [0] 结束今天          │
│ [1] 旅店老板                                │                    │
│ [2] 铜币                                    │                    │
├────────────────────────────────────────────┴────────────────────┤
│ 事件日志                                                         │
└─────────────────────────────────────────────────────────────────┘
```

规则：

- 左侧主区域放房间、出口和目标。
- 右侧只放角色状态和通用操作。
- 底部事件日志横跨全宽。
- 目标菜单使用小型上下文弹窗，不推动房间面板高度。

### 窄屏

窄屏指终端宽度低于 100 列。窄屏按阅读顺序堆叠：

```text
┌────────────────────────────┐
│ 角色状态                    │
│ 当前地点                    │
│ 房间名                      │
│ 房间描述                    │
│ 出口                        │
│ [N] 北  [E] 东              │
│ 眼前                        │
│ [1] 旅店老板                │
├────────────────────────────┤
│ 事件日志                    │
├────────────────────────────┤
│ 行动                        │
│ [R] 休息 [Q] 状态            │
│ [I] 背包 [0] 结束今天        │
└────────────────────────────┘
```

规则：

- 房间和出口仍然优先。
- 24 行高度的常见终端使用紧凑布局：顶部状态栏不加边框，行动栏放到底部，房间和日志优先可见。
- 通用操作放到底部，但按键不变。
- 窄屏可以收起详细需求条，避免挤掉房间和日志。
- 目标动作弹窗可以覆盖房间面板的一小块，但不能覆盖出口和日志。

## 菜单规则

### 目标菜单

目标菜单是小型上下文弹窗：

```text
┌─ 旅店老板 ─────┐
│ [1] 交谈        │
│ [2] 观察        │
│ Esc 返回        │
└────────────────┘
```

规则：

- 只显示当前目标的可用动作。
- 不跳新页面。
- 不使用全屏弹窗。
- 不改变主布局高度。
- `Esc` 关闭。
- 执行动作后自动关闭。
- 只用于短动作菜单。

### 背包

背包是独立流程：

- `I` 打开或关闭背包。
- 先选物品，再选动作。
- 物品动作仍然遵循目标优先。
- `Esc` 关闭背包。
- 背包打开时，不处理房间目标选择。

### 对话

对话是独立流程：

- 对话选项由数字键选择。
- `Esc` 关闭当前对话选项。
- 选择后关闭对话面板，等待回复。
- 玩家选择和 NPC 回复都进入事件日志。

### 等待中

等待中是临时状态：

- 显示轻量提示。
- 阻止重复提交。
- 不隐藏地点、出口和日志。

## 按键规则

| 按键 | 作用 | 状态 |
|------|------|------|
| `1-9` | 选择目标或菜单动作 | 根据当前状态解释 |
| `N/S/E/W/U/D` | 移动 | 普通浏览时可用 |
| `R` | 休息 | 普通浏览时可用 |
| `Q` | 状态 | 普通浏览时可用 |
| `I` | 背包 | 普通浏览或背包中可用 |
| `0` | 结束今天 | 普通浏览时可用 |
| `Esc` | 返回或关闭当前面板 | 临时状态中可用 |
| `Cmd+C` | 复制选中文字到剪切板 | 有选区时可用 |
| `Ctrl+C` | 退出客户端 | 始终可用 |

## 状态优先级

按键解释按以下优先级处理：

1. 等待中
2. 背包
3. 对话
4. 已选中目标
5. 移动方向
6. 通用操作
7. 目标选择

高优先级状态存在时，低优先级状态不响应。

## 反馈规则

- 移动后：刷新房间，并写入移动日志。
- 拾取后：目标列表更新，并写入获得日志。
- 使用后：状态变化或失败原因写入日志。
- 观察后：观察结果写入日志。
- 对话后：玩家选择和 NPC 回复写入日志。
- 失败时：错误原因写入日志，不静默失败。

## 禁止事项

- 不要把所有动作拍平成一个长列表。
- 不要在未选中目标时显示“交谈”“拾取”等目标动作。
- 不要让目标菜单遮住事件日志。
- 不要隐藏出口。
- 不要让选中目标导致主布局跳动。
- 不要让同一个按键在相同状态下产生不同含义。
