# TUI 样式规范

本文定义终端客户端的视觉风格。它只约束界面观感，不改变游戏规则，也不替代 `tui-interaction.md`。

## 风格定位

TUI 的风格是：

> 暗色中世纪文字冒险。深底、铜边、旧金焦点、暖白正文。

它应该像一个可玩的 MUD 游戏界面，而不是普通命令行工具或管理后台。

## 视觉原则

- 房间面板是第一视觉中心。
- 出口必须比普通正文更容易扫到。
- 当前焦点使用旧金色，不使用刺眼高亮。
- 事件日志像冒险记录，颜色服务于识别，不做花哨装饰。
- 侧栏克制，不抢房间主视觉。
- 弹窗小而明确，只强调当前目标。

## 面板层级

### 顶部状态栏

用途：

- 显示角色名
- 显示日期
- 显示连接状态
- 提示退出方式

样式：

- 使用主边框或次级边框。
- 背景不应比房间面板更亮。
- 连接状态用一个固定宽度符号表示，不使用长句解释。

### 房间面板

用途：

- 房间名
- 房间描述
- 出口
- 眼前目标
- 目标动作弹窗

样式：

- 使用主边框。
- 标题使用旧金色。
- 房间名比描述更亮。
- 描述使用暖白正文。
- 出口使用青绿色。

### 侧栏

用途：

- 角色需求
- 通用操作

样式：

- 使用次级边框。
- 需求条可以有颜色，但不能过亮。
- 通用操作使用固定颜色。
- 不在侧栏放目标动作。

### 事件日志

用途：

- 操作反馈
- 对话
- 系统提示
- 世界事件

样式：

- 使用次级边框。
- 背景略暗，像记录区域。
- 日志项使用固定前缀列。
- 前缀和正文可以同色，也可以前缀更亮。

### 目标动作弹窗

用途：

- 显示当前目标的短动作菜单。

样式：

- 使用焦点边框。
- 背景比房间面板略亮。
- 标题为目标名。
- 宽度固定或受屏幕约束。
- 不使用全屏遮罩。

### 背包与对话面板

用途：

- 承载内容较多的独立流程。

样式：

- 背包使用主边框。
- 对话使用焦点边框。
- 面板出现时不能遮住事件日志。
- 文案短，避免工具化说明。

## 标题规则

标题使用短词，不写解释：

- `当前地点`
- `眼前`
- `出口`
- `事件日志`
- `角色状态`
- `行动`
- `背包`
- `对话`
- `行动中`

可以使用 `◆` 作为标题前缀，但不要每一行都加装饰符。

## 按钮和快捷键

按钮以快捷键开头：

```text
[R] 休息
[Q] 状态
[I] 背包
[0] 结束今天
```

规则：

- 快捷键大写显示。
- 禁用状态使用禁用色。
- 不可用的目标动作不显示。
- 鼠标点击是辅助，键盘仍是主交互。

## 禁止事项

- 不要使用大面积纯灰。
- 不要使用后台表格感布局。
- 不要使用大面积紫蓝渐变。
- 不要为了装饰牺牲文字可读性。
- 不要使用宽度不稳定的 emoji 作为主要 UI 符号。
- 不要让侧栏比房间面板更显眼。
