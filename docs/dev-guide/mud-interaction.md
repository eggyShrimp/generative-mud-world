# MUD 交互规范

> 基于 DikuMUD / CircleMUD / Evennia / Aardwolf / BatMUD / Discworld MUD 的成熟模式。
> 修改 TUI 交互前必须阅读本节。

## 核心原则

1. **房间为中心**：房间描述永远是主视觉。物件列表、出口信息围绕房间展开，不抢占主区域。

2. **目标优先（Target-First）**：先选目标，再看可用操作。不要把所有操作拍平成一个列表。
   ```
   ✓ 正确: [1]旅店老板 → [交谈] [观察] [交易]
   ✗ 错误: [1]交谈 [2]观察 [3]交易 [4]拾取铜币 [5]拾取草药 [6]往北 [7]往南
   ```

3. **上下文敏感**：只显示当前可用的操作。没有 NPC 在场就不显示"交谈"，没有物品就不显示"拾取"。

4. **渐进披露**：第一层只显示目标和通用操作。选中目标后才展开具体动作。不要一次性展示所有可能性。

5. **出口始终可见**：方向导航是最高频操作，永远显示在显眼位置，不隐藏在子菜单里。

6. **即时反馈**：每个操作必须有可见反馈。操作结果、错误信息、状态变化——全部进事件日志。

7. **一致性**：所有交互对象（NPC、物品、出口）使用相同的"选中 → 操作"模式，不要为不同类型发明不同的交互范式。

## TUI 布局规范

```
┌──────────────┬──────────────┐
│  场景描述     │  通用操作     │
│              │  [R] 休息     │
│  可交互物件:  │  [Q] 状态     │
│  [1] NPC名   │  [I] 背包     │
│  [2] 物品名   │  [0] 结束     │
│              │              │
│  出口:        │              │
│  [N]北 [S]南  │              │
├──────────────┴──────────────┤
│  事件日志                    │
└─────────────────────────────┘
```

- **左侧/主区域**：场景描述 + 物件列表 + 出口
- **右侧/侧栏**：通用操作（不依赖物件的操作）
- **底部**：事件日志（滚动，最新在底部）

## 交互流程

```
玩家看到房间
  → 看到物件列表 [1] [2] 和出口 [N] [S]
  → 按 [1] 选中物件
    → 出现子菜单: [交谈] [观察] [取消]
    → 按对应键执行
    → 子菜单关闭，结果显示在事件日志
  → 按 [N] 直接移动
  → 按 [R] 执行通用操作
```

## 按键约定

| 按键 | 作用 | 上下文 |
|------|------|--------|
| `1-9` | 选中物件/目标 | 物件列表可见时 |
| `N/S/E/W` | 移动 | 始终可用 |
| `R/Q/I` | 休息/状态/背包 | 始终可用（通用操作） |
| `0` | 结束今天 | 始终可用 |
| `Esc` | 关闭子菜单/返回 | 子菜单打开时 |
| `Tab` | 设置 | 始终可用 |

## 禁止事项

- 不要把所有操作拍平成一个长列表
- 不要在没有选中目标时显示"交谈""拾取"等具体操作
- 不要让子菜单覆盖事件日志
- 不要隐藏出口信息
- 不要在操作执行后不给反馈
